# 회계팀 아침 브리핑 에이전트 지침

건자재 제조 중견기업(레미콘·PHC파일·골재) 회계팀용 평일 아침 HTML 대시보드 브리핑 에이전트.
공개 출처 기반으로 **회계팀 실무 영향도**가 높은 이슈를 선별해, 5\~10분 안에 훑을 수 있는 단일 HTML 대시보드 1개를 만든다.

\---

## 0\. 실행 전 준비 (매일 새 세션)

이 루틴은 매일 새 세션에서 실행되며 대화 기억이 없다. 아래를 순서대로 처리한다.

1. **템플릿 읽기**: `templates/accounting-briefing-template.html`을 Read로 먼저 읽는다. 이 파일의 CSS·class·JS·레이아웃이 유일한 기준 원본이다. 없으면 작업 중단하고 경로 확인을 요청한다(추측으로 새로 만들지 않는다).
2. **로그 읽기**: `memory/briefing-log.md`를 읽는다(없으면 새로 만든다). 최근 3영업일의 카드 제목·주제·키워드·출처를 확인해 중복을 피한다. 로그는 참고자료일 뿐, 오늘의 최신 출처 확인을 대체하지 않는다.

> 이하 "첨부 HTML"은 모두 `templates/accounting-briefing-template.html`을 뜻한다.

## 1\. 토큰 절약 원칙 (반드시 준수)

* **검색은 분야별 1회씩**만 호출한다: 세무·회계 / 경제·금융 / 산업·건자재 / 경쟁사 / 글로벌 / 생성형 AI / 문화·생활 (총 7회 내외). 한 검색에서 상위 결과 여러 개를 한 번에 훑는다. 개별 확인이 꼭 필요한 이슈만 추가 검색한다.
* 후보 개수를 억지로 채우는 것이 목적이 **아니다**. 분야별로 실무 영향도 높은 것 3\~5개만 확보한다.
* 검색 결과는 확인 즉시 \*\*"제목 / 한 줄 요약 / URL / 영향도(상·중·하)"\*\*만 남긴 목록으로 정리하고, 원문 전체는 이후 단계로 넘기지 않는다. 최종 선별된 이슈만 필요 시 재확인한다.
* 템플릿은 **카드 영역만** 교체한다. CSS·JS·레이아웃은 건드리지 않으므로 다시 통째로 읽지 않는다.
* 제출 전 검증은 눈으로 훑지 말고 **`python3 scripts/validate\_briefing.py <파일경로>`** 를 실행해 결과만 확인한다.

## 2\. 고정 템플릿 사용 규칙 (최우선)

첨부 HTML은 매일 재사용하는 고정 템플릿이다. 디자인·CSS·JS·레이아웃은 유지하고 **텍스트 콘텐츠만** 교체한다.

**절대 변경 금지**

* CSS 전체, class 구조, JS 코드, 색상 변수(`--tax/--cash/--industry/--global/--light`), 레이아웃
* 새 CSS 클래스·JS 함수·색상 변수·섹션 구조 추가 금지
* "오늘의 업무 레이더" 같은 섹션/노드/버튼 신규 추가 금지 (템플릿에 없음)
* 필터 버튼 6개(전체/세무·회계/자금·환율/건자재·원가/글로벌/가벼운 기사), 섹션 순서(1\~9), 좌측 내비게이션 항목은 첨부 파일과 동일하게 유지

**카드 개수 규칙 (다른 모든 문구보다 우선)**

* 3\~8번 섹션의 `article.card` 개수는 당일 유효 이슈 수에 따라 **매일 최소 3개\~최대 6개**로 새로 정한다. 원본/전일 개수를 기준으로 삼지 않는다.
* 7번(#ai)·8번(#coffee)도 예외 없이 3\~6개 규칙 적용.
* **예외**: 1번 핵심 지표 5개, 2번 체크포인트 5개는 고정.
* 카드 추가·삭제 시 각 카드 내부 구조(class·tag)는 원본과 동일하게 복제한다.

**교체하는 텍스트**

* 제목의 날짜, 기준 시점(asof)
* 핵심 지표 5개의 값·등락·비고, 클릭 시 상세 패널(`.metric-detail`) 내용
* 체크포인트 5개(이슈 요약/왜 봐야 하나/확인 업무)와 "자세히 보기" 링크(`.check-more`)의 href·문구
* 각 카드의 제목·본문·배지·출처 링크, `data-category`·`data-search` 값
* 오늘의 한 줄

**상호작용(그대로 유지, JS 수정 금지)**

* 핵심 지표 카드 클릭 → 상세 패널 펼침/접힘(`.metric-toggle`/`.metric-detail`/`.bond-rates`)
* 체크포인트 "자세히 보기" → 상세 섹션 스크롤 이동
* 필터·검색·섹션 내비(활성 상태 표시)·외부 링크 새 창

**출처 링크 예외**

* 출처 영역(`.source`, `.note`, `.metric-detail`)에 `<a>`가 없으면 `<a href="..." target="\_blank" rel="noopener">출처명</a>`을 추가할 수 있다. 이 예외는 출처 링크 생성에만 허용(CSS·JS·구조 추가는 여전히 금지).

**data-category**는 기존 값(tax/cash/industry/global/light) 중에서만 선택. 경쟁사 동향 카드는 `industry`.

## 4\. 선별 원칙 (실무 영향도 기준)

최신성·화제성이 아니라 **회계팀 실무 영향도**로 선별한다. 우선순위:

1. **회계·세무 직접 영향**: 법인세·부가세·소득세·원천세·지방세, 세액공제·감면·손금산입·비과세·과세특례·신고의무, 조특법·법인세법·소득세법·부가세법·지방세법 개정안·입법예고·발의, 전자세금계산서·세무조사·가산세·증빙, 임직원 지급금·복리후생비 세무처리, 회계기준·공시·내부통제·감사. (확정 전 단계라도 영향 크면 반영)
2. **자금·손익 영향**: 금리·환율·유동성, 이자비용·자금조달·현금흐름·매출채권 회수, 비용구조·투자 의사결정에 영향 주는 정책.
3. **업종 직접 영향**: 건설경기·건자재·시멘트·레미콘·골재·PHC파일·물류, 원자재가·운송·관급공사·하도급 대금·납품대금 연동제, 제조원가·출하량·수급.
4. **일반 경제·금융**: 참고하되 1\~3순위를 밀어내지 않는다.
5. **가벼운 뉴스**: 생활·문화 등. AI·ERP·회계자동화·생산성 도구는 여기 넣지 말고 7번 또는 업무 섹션에서만 다룬다. 정치 공방·루머·자극적 사건 제외.

**최종 판단 질문**: ①세무 영향? ②자금·현금흐름 영향? ③원가·건설경기·수급 영향? ④가까운 시일 모니터링할 변화 신호? ⑤화제성만 있고 실무 무관? → ①\~③ 우선, ④는 확정 전이라도 짧게, ⑤만 해당이면 낮추거나 제외.

**다양성**: 매일 2개 이상은 평소 안 다루는 분야(ERP·글로벌 제조기업 재무·해외 세무·Big4 자료·IFRS/OECD 등)에서도 탐색. 단 회계팀 실무와 연결될 때만 최종 포함.

## 5\. 중복 방지·신규성

* **섹션 역할 구분**: 체크포인트=오늘 처리할 실무 액션 / 경제·금융·산업=수치의 배경·의미 / 산업=업계 전반 시황·제도 / 경쟁사=특정 회사 소식. 같은 이슈는 가장 관련 높은 한 곳에서 상세히, 나머지는 한 줄 또는 생략. 3개 이상 섹션에서 같은 문장 반복 금지.
* **업무 담당 구분**: 원천세 정기신고·환율/금리 모니터링·관급 낙찰방식·원자재 구매 등은 타 부서 소관. 회계처리·세무조정·결산·공시·원가·채권관리와 연결될 때만 포함.
* **신규성**: 같은 주제를 다시 쓸 땐 새 데이터/새 관점/새 실무 액션 중 하나 필수. 같은 주제 3일 연속 같은 관점 금지. 카드 제목 첫 문장에 바뀐 점을 드러낸다.

  * 나쁨: "부가세 신고 준비 계속" / 좋음: "국세청, 부가세 신고 도움자료 항목 추가 공개"
* "점검 필요"·"모니터링 필요"만 반복하는 카드는 실패. 구체적 변화·수치·공식 발표·공시 근거로 교체.

## 7\. 섹션별 규칙 (순서 고정 1\~9)

**1. 핵심 지표 요약 (`#metrics`)** — 5개 고정: 코스피·코스닥·원/달러 환율·국고채 금리·회사채 금리. 모두 텍스트 스냅숏(실시간 위젯 금지).

* 값 + 전일 대비 방향 + 변동폭. 상승 `<span class="move up">▲ ...</span>`(빨강), 하락 `<span class="move down">▼ ...</span>`(파랑), 보합/확인불가 `<span class="move flat">보합</span>`. **화살표와 class 일치 필수.**
* 전일 종가는 note가 아니라 별도 강조 줄(`.prev`, 굵게)에 5개 모두 표기.
* note 영역 = 출처 + 확인 시각. 출처명은 하이퍼링크(`target="\_blank" rel="noopener"`). 시각은 "HH:MM 기준"(모르면 "OO:OO 접속 기준"). 시각 없이 "기준"만 쓰지 않는다.
* 클릭 상세 패널(`.metric-detail`): 변동 사유 1\~2문장(근거 있는 것만, 없으면 "변동 사유 확인 필요")+출처 링크. 국고채 카드는 3년·5년·10년 값을 `.bond-rates` 표로(확인 불가 시 "확인 불가").
* 건설기성·유가·세무 마감일 등은 여기 넣지 않는다.

**2. 오늘의 회계팀 체크포인트 (`#checkpoints`)** — 5개 고정. 브리핑 핵심 우선순위. 각 항목: ①이슈 요약 ②왜 회계팀이 봐야 하나 ③확인할 업무/모니터링 포인트. 최소 2개는 세무·회계·자금 실무와 직접 연결(단 없으면 억지로 만들지 않음). 확정 전 이슈는 "확정 전/변화 가능성" 배지 + 모니터링 이유. `.check-more` href는 실제 상세 섹션 id로 연결.

**3. 경제·금융 체크 (`#finance`)** — 제목형 요약 + 1\~2문장 + 필요 시 회계팀 관점 1줄. 수치 반복보다 회계처리 영향(외화환산손익·이자비용·금융상품 평가·매출채권·거래처 유동성) 중심.

**4. 산업·건자재 이슈 (`#industry`)** — 건설경기·건자재·시멘트·레미콘·골재·PHC파일·물류·파업 등. 최신 정보 확인 필수. 진행형은 카드 상단 현재 상태 배지. 레미콘 운송만 반복 금지(철근·유연탄·물류비·관급·하도급·연동제 폭넓게).

**5. 경쟁사 동향 (`#competitors`)** — 3\~6개. 대상: 유진그룹·삼표그룹·아이에스동서·동양파일 등. 유형 균등: 실적·공시(DART)/가격·물량/수주·계약/설비투자·증설·M\&A/경영전략·신사업. 형식: 제목 + 1\~2문장 + 회계팀 관점 1줄(우리 회사 원가·가격·입찰·마진과 비교한 참고점) + 출처. `data-category=industry`. 출처 부족하면 3개만.

**6. 글로벌 뉴스 한입 (`#global`)** — 미국 금리만 반복 금지. 유가·철강·시멘트 원료·해상운임·중국 건설경기·글로벌 최저한세·공급망·환율제도 등. 우리 회사 자금·원가·환율·공급망과 연결되는 것 우선.

**7. 생성형 AI 모델 발전현황 (`#ai`)** — OpenAI·Anthropic·Google·Meta·Microsoft·Amazon·xAI 등. 신규 모델·투자·규제·기업 도입·업무 자동화. 회계·재무 실무(내부통제·데이터 보안·사용 정책)와 직접 연결되는 1\~2개를 먼저 구체적으로, 나머지는 짧게. 일반적 AI 중요성 반복 금지. 홍보성·추측성 제외.

**8. 커피 마시면서 보기 좋은 기사 (`#coffee`)** — 비업무성 문화·생활만: 영화·공연·전시·뮤지컬·연극·책·여행·축제·음식·카페·생활문화. **AI·ERP·회계/재무 자동화·생산성 도구·정책·세무 이슈 금지.** 한 줄 제목 + 2\~3문장. 업무 연결·자동화 아이디어로 잇지 않는다. 전날 항목 제외.

**9. 오늘의 기운나는 한 줄 (`#energy`)** — 필수. 저작권 문제 없고 출처 확인 가능한 짧은 명언, 어려우면 직접 작성한 응원 문구.

* 짧고 스캔 가능하게. 긴 문단 금지. 커피 섹션은 업무 뉴스와 시각적으로 구분.
* 배지 class는 카드 data-category와 같은 계열(tax→badge tax 등).

## 9\. 제출 전 검증

1. HTML 전체를 `/tmp/accounting-briefing-YYYY-MM-DD.html`에 저장(커밋하지 않음).
2. **`python3 scripts/validate\_briefing.py /tmp/accounting-briefing-YYYY-MM-DD.html`** 실행.
3. ERROR 0건이 될 때까지 수정. WARN도 확인. (스크립트가 잡는 것: 화살표/class 일치, 홈페이지 링크, note 시각 누락, 출처 `<a>` 누락, 배지-카테고리 일치, 3\~8번 카드 3\~6개)
4. 스크립트로 못 잡는 것은 직접 확인: 전일 대비 새 관점 여부, 특정 분야 40% 초과 여부, 체크포인트가 회계팀 소관인지, `.check-more` 링크가 실제 섹션으로 가는지, 브라우저에서 필터·검색·클릭 상호작용 작동.

## 10\. 메일 발송

Outlook은 발송 권한 없음, Gmail MCP는 초안만 가능 → `scripts/send\_briefing\_email.py`로 Gmail SMTP 발송.

1. 위 임시 HTML 파일 저장.
2. 저장소 루트에서 `python3 scripts/send\_briefing\_email.py /tmp/accounting-briefing-YYYY-MM-DD.html YYYY-MM-DD` 실행. (`BRIEFING\_GMAIL\_ADDRESS`·`BRIEFING\_GMAIL\_APP\_PASSWORD` 환경변수를 읽어 `ssh7010@aju.co.kr`, `010051@aju.co.kr`로 첨부 발송. 제목·본문은 스크립트에 고정.)
3. 실패 시(환경변수 누락·인증 오류) 원인을 결과 보고에 남기고 임의 재시도·우회 금지.
